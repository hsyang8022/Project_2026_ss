# 기능명세서 및 유저 시나리오 최종 수정 액션플랜

## 0. 최종 수정 방향

본 기능명세서는 **AI 자동판단 서비스**가 아니라, **제한된 공개 데이터셋 기반의 EO/SAR 위성영상 판독 보조 서비스**로 정리한다.

핵심 방향은 다음과 같다.

- 공식 사용자는 **영상판독관**과 **지휘관**으로 한정한다.
- AI 결과는 **표적 확정값**이 아니라 **표적 후보**로만 제공한다.
- Grad-CAM은 판독관용 **AI 집중 영역 검토 기능**으로 사용한다.
- SHAP은 메인 판독 기능이 아니라 **내부 검증 또는 향후 고도화 기능**으로 분리한다.
- 지휘관 화면은 **타격 지시**가 아니라 **조치 기록·상급 보고·대응 준비 판단 지원** 중심으로 수정한다.
- 분석 로그는 **AI 결과 / 판독관 판단 / 지휘관 조치 기록**을 분리 저장한다.

---

## 1. 사용자 유형 재정리

### 현재 문제

현재 사용자 유형이 다음 4개로 구성되어 있음.

- 영상판독관
- 지휘관
- 데이터 사이언티스트
- 관리자 / 유지보수

이는 서비스 범위를 넓혀 **판독 보조 서비스**가 아니라 **모델 운영 플랫폼**처럼 보일 수 있음.

### 수정 액션

| 구분 | 수정 방향 |
|---|---|
| 영상판독관 | 공식 사용자로 유지 |
| 지휘관 | 공식 사용자로 유지 |
| 데이터 사이언티스트 | 공식 사용자에서 제외, 내부 검증 또는 향후 고도화 항목으로 이동 |
| 관리자 / 유지보수 | 공식 사용자에서 제외, 운영 관리 항목 또는 부록으로 이동 |

### 반영 문구

> 본 서비스의 주요 사용자는 영상판독관과 지휘관으로 한정한다. 모델 성능 검토, SHAP 분석, 시스템 유지보수 기능은 본 서비스의 핵심 사용 시나리오가 아닌 내부 검증 또는 향후 고도화 기능으로 분리한다.

### 우선순위

**최상**

## 2. 서비스명 수정

### 현재 문제

현재 서비스명은 다음과 같음.

> EO/SAR 영상 기반 지상 표적 자동탐지 및 분석 지원 서비스

하지만 EO 분석 대상에 항공기, 헬리콥터 등이 포함되어 있어 **지상 표적**이라는 표현과 일부 충돌함.

### 수정 액션

서비스명을 다음 중 하나로 수정한다.

### 1안: 가장 안전한 명칭

> EO/SAR 위성영상 기반 표적 후보 탐지 및 판독 지원 서비스

### 2안: 기존 표현을 최대한 유지한 명칭

> EO/SAR 위성영상 기반 지상·공중자산 후보 탐지 및 분석 지원 서비스

### 추천안

> **EO/SAR 위성영상 기반 표적 후보 탐지 및 판독 지원 서비스**

### 수정 이유

- 지상 장비, 항공기, 헬리콥터를 모두 포괄 가능
- AI가 표적을 확정한다는 인상을 줄임
- 판독관 중심 서비스 목적과 일치

### 우선순위

**높음**

## 3. 타격 관련 표현 완화

### 현재 문제

현재 기능명세서에 다음 표현이 포함되어 있음.

- 타격 지시
- 타격 준비
- 피해 평가 완료
- 파괴 흔적 확인

이 표현들은 서비스가 군사적 타격 판단을 직접 수행하거나, 타격 효과를 확정 평가하는 것처럼 보일 수 있음.

### 수정 액션

| 기존 표현 | 수정 표현 |
|---|---|
| 타격 지시 | 결심 사항 기록 |
| 타격 준비 | 대응 준비 |
| 추가 정찰 | 추가 정찰 요청 |
| 피해 평가 완료 | 사후 변화 확인 |
| 파괴 흔적 확인 | 표적 후보 변화 여부 확인 |
| 타격 후 변화 분석 | 사후 재판독 및 변화 기록 |

### 지휘관 조치 선택지 수정

#### 기존

- 지속 감시
- 추가 정찰
- 타격 준비
- 타격 지시

#### 수정

- 지속 감시
- 추가 정찰 요청
- 대응 준비
- 상급 보고 / 결심 사항 기록

### 반영 문구

> 지휘관은 판독관이 검토한 표적 후보 정보를 바탕으로 지속 감시, 추가 정찰 요청, 대응 준비, 상급 보고 등 후속 조치를 기록할 수 있다. 본 서비스는 자동 타격 권고나 화력유도 자동화 기능을 제공하지 않는다.

### 우선순위

**최상**

## 4. Grad-CAM / SHAP 기능 범위 조정

### 현재 문제

현재 안에는 다음 표현이 있음.

> 시스템이 두 기법의 집중 영역을 자동 비교하여 태그 부여 및 분석 로그 저장

이는 구현 난도가 높고, 기술적으로 질문을 받을 가능성이 큼.

### 수정 액션

| 항목 | 수정 방향 |
|---|---|
| Grad-CAM | 판독관용 메인 기능으로 유지 |
| SHAP | 메인 기능에서 제외하고 내부 검증 또는 향후 고도화 기능으로 이동 |
| 자동 비교 | 반자동 참고 태그 제안 방식으로 수정 |
| 자동 태그 | 판독관이 최종 확인·수정 가능하도록 변경 |
| Worst-Case 대응 로직 | XAI 결과 검토 가이드로 표현 완화 |

### 수정 문구

> Grad-CAM은 판독관이 AI가 주로 참고한 영역을 직관적으로 확인하기 위한 AI 집중 영역 검토 기능으로 제공한다. SHAP은 모델 판단 특성을 추가 분석하기 위한 내부 검증 또는 향후 고도화 기능으로 분리한다.

> XAI 결과는 자동 확정 기준이 아니라, BBox와 AI 집중 영역의 중첩 여부를 참고하여 시스템이 재검토 필요 후보를 제안하는 방식으로 활용한다. 최종 판단과 태그 수정은 영상판독관이 수행한다.

### 우선순위

**최상**

## 5. XAI 불일치 처리 규칙 수정

### 현재 문제

현재는 Grad-CAM과 SHAP을 동등하게 비교하는 구조임.  
그러나 두 기법은 설명 방식이 달라 결과가 다르게 나올 수 있음.

### 수정 액션

Grad-CAM 중심으로 단순화하고, SHAP 비교는 본문에서 축소한다.

### 수정 전

| 상황 | 서비스 화면 표시 | 판독관 대응 가이드 |
|---|---|---|
| Grad-CAM과 SHAP 일치 | 근거 일관성 높음 | 신뢰 가능 |
| Grad-CAM은 표적, SHAP은 배경 | 재검토 필요 | 육안 재확인 |
| Grad-CAM은 배경, SHAP은 표적 | AI 근거 불안정 | 판단 보류 |
| 둘 다 배경 | 오탐 의심 | 제외 권장 |

### 수정 후

| 상황 | 서비스 화면 표시 | 판독관 대응 가이드 |
|---|---|---|
| Grad-CAM 집중 영역이 표적 후보 박스 내부에 주로 위치 | AI 집중 영역 일치 | 원본 영상과 함께 우선 검토 |
| Grad-CAM 집중 영역이 표적 후보와 배경에 혼재 | 재검토 필요 | 원본 영상, Crop 이미지, 주변 맥락 재확인 |
| Grad-CAM 집중 영역이 표적 후보 외부에 주로 위치 | 오탐 의심 | 보류 또는 제외 검토 |
| Grad-CAM 집중 영역이 넓게 분산 | 근거 불명확 | 추가 확인 또는 판독 보류 |

### SHAP 처리 문구

> SHAP 결과는 판독관용 메인 화면에 직접 노출하지 않고, 모델 취약 조건을 검토하기 위한 내부 분석 자료로 활용한다.

### 우선순위

**높음**

## 6. Gate-Keeping 조건 구체화

### 현재 문제

현재 Gate 조건이 모호함.

- AI 신뢰도 등급화 정합성 80%
- 오류 분별 알고리즘 검증 완료
- PDF 출력 포맷 안정화

특히 “정합성 80%”와 “오류 분별 알고리즘”은 정의가 불분명함.

### 수정 액션

Gate 조건을 성능 기준이 아니라 **구현 완료 기준**으로 변경한다.

### 수정안

```text
[PoC 단계]
- EO/SAR 테스트 영상 입력
- AI 모델 추론 결과 출력
- 표적 후보 박스, 추정 클래스, AI 신뢰도 수치 확인

Gate 1
- EO/SAR 테스트 영상 입력 시 표적 후보 박스, 추정 클래스, AI 신뢰도 등급이 화면에 정상 표시될 것

[MVP 단계]
- 판독관 검토 화면 구현
- Grad-CAM 기반 AI 집중 영역 표시
- 판독 상태 및 코멘트 저장
- 분석 로그 누적 관리

Gate 2
- 선택 객체에 대해 AI 집중 영역이 표시되고, 판독관 상태값과 코멘트가 저장될 것

[최종 구현 단계]
- 지휘관 요약 화면 구현
- 분석 로그 기반 자동 보고서 생성
- PDF 또는 인쇄 서식 Export
- 사후 재판독 기록 저장

Gate 3
- 저장된 분석 로그를 기반으로 지휘관 요약 보고서가 PDF 또는 인쇄 서식으로 정상 출력될 것
